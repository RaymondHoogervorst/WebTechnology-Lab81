Products List API

This is the manual for the Max Market database API, used to access the Max Market products list.

Retrieve full data set

URL:
http://localhost:3000
Method:
GET
Response type:
JSON
Description:
Returns a json object containing all products from the database.
Each product is an object with the following properties:
  • ID
  • name
  • origin
  • best_before_date
  • amount
  • image

Add data for new product item

URL:
http://localhost:3000
Method:
POST
Request type:
JSON
Response type
text
Description:
Adds the product described in JSON to the database and returns the assigned ID.
The properties that need to be defined in the request JSON are:
  • name
  • origin
  • best_before_date
  • amount
  • image

Retrieve certain product

URL:
http://localhost:3000/{productID}
Method:
GET
Response type:
JSON
Description:
Returns a json object containing the product with the id given in the URL.
The object returned has the following properties:
  • id
  • name
  • origin
  • best_before_date
  • amount
  • image

Update certain product

URL:
http://localhost:3000/{productID}
Method:
PUT
Request type:
JSON
Response type:
None
Description:
Edit data of an existing product with the id given in the URL.
The properties that can be defined in the request JSON are:
  • name
  • origin
  • best_before_date
  • amount
  • image
All of these properties are optional. The properties only need to be defined if their corresponding field needs updating.

Remove certain product

URL:
http://localhost:3000/{productID}
Method:
DELETE
Request type:
Text
Response type:
None
Description:
Deletes the product with the id given in the URL.

Response codes

  • 200 The request has been succesfully recieved.
  • 201 The request has been succesfully recieved and the database has been adjusted.
  • 400 At least one of the required attributes is not found in the request JSON.
  • 403 The method you are trying to use is unavailable.
  • 404 The ID of the product you tried to reach could not be found.
  • 500 There is an internal error on serverside
